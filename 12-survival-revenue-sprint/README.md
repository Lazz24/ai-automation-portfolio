## What It Does

A structured sprint framework for achieving minimum viable revenue under tight time and budget constraints. The model targets 50 EUR output per 4 hours of work per week, with a revenue-first prioritisation filter that cuts any activity not on the direct path to income. Includes tactical pivot logic for adjusting when the current approach is not converting.

---

## System Components

| Component | Description |
|-----------|-------------|
| Sustainability model | 50 EUR / 4hr/week target as baseline |
| Lead acceleration | Structure for compressing lead-to-close cycle |
| Revenue-first filter | Decision framework — cut anything not directly revenue-generating |
| Tactical pivot logic | Trigger conditions and pivot options when conversion stalls |

---

## Key Design Decisions

- Revenue-first filter is applied before any other prioritisation — time is the primary constraint
- Tactical pivot logic has defined trigger conditions — avoids reactive pivoting without signal
- Model is designed to be repeatable week-over-week, not just a one-time sprint
